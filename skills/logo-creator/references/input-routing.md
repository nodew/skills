# Input Routing

Use this reference to decide how much clarification is necessary before proposing directions.

## Input Types

### Text brief

Signals:

- app or brand name
- market or audience description
- style words like `minimal`, `playful`, `premium`, `technical`
- color preferences or constraints

If the brief is usable, move forward after filling only critical gaps.

### Reference images

Signals:

- image paths
- screenshots of other logos or mood boards
- sketches or icons the user wants to react to

Treat references as style signals, not templates to reproduce.

### Mixed input

Signals:

- brief plus one or more image paths
- existing logo plus a redesign request

Use the brief to define business context and the references to calibrate visual direction.

## Mixed Clarification Rule

Ask only for missing information that would materially change the result.

High-priority gaps:

- brand or product name
- what the product does
- intended audience or tone
- whether the logo must function as an app icon

Low-priority gaps that should not block progress unless essential:

- detailed backstory
- marketing copy
- exhaustive competitor analysis

## Suggested Clarifying Questions

Use one focused question at a time.

Examples:

- `What exact brand or app name should appear in the wordmark?`
- `Should this logo primarily optimize for app-icon usage, website header usage, or both?`
- `Do you want the logo to feel more technical, friendly, or premium?`

## Route Outcomes

- If context is strong enough: propose 2-3 directions immediately.
- If context is weak but recoverable: ask for the smallest missing set, then continue.
- If the user only wants export help for an existing SVG: skip direction work and go straight to packaging.
